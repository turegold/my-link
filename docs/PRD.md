# 제품 요구사항 정의서 (PRD) - 마이링크 (MyLink)

**문서 정보**
- **버전:** v0.4
- **작성일:** 2026-03-20
- **상태:** 초안 (Draft)

---

## 1. 프로젝트 개요
- **프로젝트명:** 마이링크 (MyLink)
- **목적:** 링크트리(Linktree)의 핵심 기능을 구현한 클론 서비스로, 사용자가 본인의 여러 웹사이트 링크를 URL 슬러그 주소를 통해 공유할 수 있게 해줍니다.
- **대상 사용자:** 본인의 여러 링크를 하나의 페이지로 관리하고 싶은 누구나
- **수익화 방향:** 예정 없음 (전 기능 무료)
- **디자인/UI:** `shadcn/ui` 기반 모던 UI 적용 (테마 커스텀 기능 제외)

---

## 2. 핵심 기능 목록

### 2.1. 필수 기능 (MVP)
- **사용자 인증 (Firebase 기반)** 
  - 구글 소셜 로그인만 지원
- **URL 슬러그 초기화 및 자동 구성 (온보딩)** 
  - 구글 로그인 시, 지메일 아이디의 앞 부분(예: `abc@gmail.com` -> `abc`)을 가져와서 **`displayName`** 값으로 자동 설정
  - 이 `displayName` 값이 사용자의 고유 접속 URL 슬러그(`domain.com/[displayName]`)로 작동
- **링크 관리 (CRUD)**
  - 타이틀과 URL로 이루어진 링크 데이터 생성, 조회, 수정, 삭제
  - 텍스트를 클릭하여 그 자리에서 내용을 수정하는 **인라인 편집(Inline Editing)** 적용
  - **구글 파비콘 API**를 통해 등록된 URL 사이트의 파비콘을 아이콘으로 렌더링
- **프로필 정보 설정**
  - URL 경로인 `displayName` 외에 화면에 보이는 실제 사용자 이름 **`username`** 별도 관리
  - 닉네임(`username`)과 자기소개(Bio) 모두 텍스트 **인라인 편집** 수정 가능
  - *(참고: 사용자가 프로필 이미지를 직접 업로드하는 기능은 없음)*
- **퍼블릭 프로필 (Public 뷰) 페이지**
  - URL 경로(`domain.com/[displayName]`) 접속 시 사용자 페이지 노출
  - 페이지 메인 화면에는 URL 경로 값이 아닌, 실제 표시용 이름인 `username` 노출

### 2.2. 추후 추가 기능
- **개별 링크 클릭 통계:** 링크 아이템별 클릭 조회수 집계 및 노출

## 3. 각 기능의 상세 설명

### 3.1. 인증 및 온보딩 (Authentication & Onboarding)
- **설명:** 번거로운 가입 절차 없이 구글 로그인 버튼 1개로 즉시 가입/로그인.
- **세부 동작:**
  - 최초 구글 계정으로 로그인 성공 시, 이메일 주소의 `@` 앞부분을 파싱하여 **`displayName`**으로 저장합니다. 이를 토대로 퍼블릭 프로필 페이지 주소(URL Slug)가 즉시 확보됩니다.
  - 동시에 구글 프로필 사진 정보를 바탕으로 `photoURL` 필드값을 취합하여 저장합니다.

### 3.2. 대시보드 - 프로필 설정 (Profile Settings)
- **설명:** 내 프로필 상단에 노출될 개인화 영역을 관리합니다.
- **세부 동작:**
  - 이미지 업로드 폼은 제공하지 않으며 구글에서 받아온 초기 프로필 이미지(`photoURL`)를 그대로 노출하거나 표시하지 않는 단순 로직으로 구현합니다.
  - 이름, 소개글 텍스트를 화면에서 마우스로 클릭하면 즉시 입력칸이 활성화되며, 내용을 수정한 후 엔터를 누르거나 외곽을 클릭하면 **인라인 편집 방식**으로 데이터베이스에 변경 사항(`username`, `bio`)을 즉시 저장합니다.

### 3.3. 대시보드 - 링크 관리 (Link Management)
- **설명:** 실제 외부로 이동하는 버튼들을 관리하는 기능.
- **세부 동작:**
  - 제목과 URL 2개만 입력받아 리스트형태로 만들어주고, 수정 시에도 **인라인 편집**으로 전환하여 편리하게 수정합니다.
  - 구글 API(`https://s2.googleusercontent.com/s2/favicons?domain=...`)를 활용해 URL의 루트 호스트 영역 기반 파비콘을 가져옵니다.

### 3.4. 퍼블릭 프로필 페이지 (Public Profile View)
- **설명:** 타인에게 전달되는 렌더링 전용 결과 페이지.
- **세부 동작:**
  - `/displayName` 매개변수 기반의 라우팅으로 접속하며, 페이지 최상단엔 사용자 지정 실명(`username`)과 한 줄 소개가 깔끔하게 노출됩니다.

## 4. 데이터베이스 및 데이터 모델링 (Firebase Firestore)

안정성과 성능 확장을 위해 유저 정보 컬렉션과, 개인이 가진 다수의 링크들을 담는 서브 컬렉션으로 분할하여 테이블을 설계합니다.

### 4.1. Users Collection (`users`)
회원 개인별 식별 및 프로필 정보를 담는 루트 문서입니다. 지메일 prefix 기반의 URL 슬러그(`displayName`)와 실제 보여질 이름(`username`)이 명확히 대비됩니다.

| 필드명 | 타입 | 설명 |
|---|---|---|
| `uid` | String | Firebase Auth 고유 식별자 (문서 ID) |
| `email` | String | 가입 된 구글 이메일 주소 |
| `username` | String | **실제 화면 상단에 보여질 사용자 이름** |
| `displayName` | String | **퍼블릭 페이지 URL Slug** (지메일 ID 앞부분으로 초기화) |
| `photoURL` | String | 구글 계정에 연동된 프로필 사진 URL |
| `bio` | String | 짧은 자기소개 안내글 |
| `createdAt` | Timestamp | 계정 최초 생성 시간 |

### 4.2. Links Sub-collection (`users/{uid}/links`)
한 유저가 생성한 여러 개의 이동 링크 정보들을 개별 문서 구조로 모아둔 서브 컬렉션입니다.

| 필드명 | 타입 | 설명 |
|---|---|---|
| `id` | String | 생성된 링크의 고유 문서 식별자 |
| `title` | String | 대시보드 및 퍼블릭 페이지에서 노출되는 링크 제목 |
| `url` | String | 사용자가 클릭했을 때 이동할 목적지 실제 주소 |
| `createdAt` | Timestamp | 해당 링크가 생성된 일시 |
