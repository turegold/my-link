# 마이링크 (MyLink) 사용자 시나리오 (User Scenario)

**문서 정보**
- **버전:** v0.1
- **작성일:** 2026-03-20
- **상태:** 초안 (Draft)

---

## 1. 일반 방문자 시나리오 (Visitor)

### 1.1. 유효한 프로필 접속 및 외부 이동
- 사용자는 **특정 사용자의 링크 모음을 한 곳에서 확인하기 위해** 소유자가 공유한 `domain.com/[displayName]` 형식의 정상 URL로 접속한다.
- 사용자는 **현재 들어온 페이지의 주인이 누구인지 명확히 파악하기 위해** 화면 최상단에 센터 정렬로 노출된 사용자 이름(`username`)과 짧은 한 줄 소개글을 확인한다.
- 사용자는 **원하는 목적지가 어느 버튼인지 식별하기 위해** 나열된 링크 버튼 항목들의 좌측에 자동으로 추출된 파비콘 이미지(아이콘)와 제목(Title) 텍스트를 함께 확인한다.
- 사용자는 **해당 목적지 웹사이트로 페이지를 이동하여 외부 콘텐츠를 소화하기 위해** 관심 있는 특정 링크 블록 영역 중 하나를 터치(클릭)한다.

### 1.2. 유효하지 않은 프로필 접속 (404 예외 처리)
- 사용자는 **오타나 탈퇴 등으로 인해 존재하지 않는 주소의 프로필 모음을 열람하기 위해** 잘못된 URL(예: `domain.com/없는유저`)로 비정상 접근한다.
- 사용자는 **현재 접속한 페이지 주소가 잘못되었음을 시스템으로부터 안내받기 위해** 화면 중앙에 렌더링 된 "존재하지 않는 페이지입니다"라는 직관적인 문구를 확인한다.
- 사용자는 **정상적인 마이링크 서비스의 메인 홈페이지 화면으로 돌아가기 위해** 오류 안내 하단에 여유롭게 제공되는 `[홈으로 돌아가기]` 유틸리티 버튼을 클릭한다.

## 2. 서비스 소유자 시나리오 (Owner)

### 2.1. 인증 및 온보딩 (URL 슬러그 초기화)
- 사용자는 **별도의 복잡한 회원가입 폼 작성 없이 서비스를 즉시 이용하기 위해** 메인 랜딩 페이지에서 제공되는 유일한 `[구글로 시작하기]` 버튼을 클릭한다.
- 사용자는 **자신의 퍼블릭 페이지 접속 경로를 최초 확보하기 위해** 구글 인증 완료를 거치고, 자신의 지메일 아이디의 앞부분(`@` 이전 부분)을 기반으로 `displayName` 값이 자동 설정되어 온보딩이 즉각적으로 통과되는 과정을 거친 뒤 대시보드 화면으로 진입한다.

### 2.2. 대시보드 진입 및 유틸리티 기능 활용
- 사용자는 **방문자 입장에서 자신의 퍼블릭 페이지가 화면에 어떻게 렌더링 되는지 즉각 확인하기 위해** 대시보드 상단에 위치한 `[내 페이지 보기]` 버튼을 클릭하여 새 탭에서 본인 프로필 페이지를 띄운다.
- 사용자는 **등록된 자신의 링크 모음을 타인에게 보내거나 SNS 바이오 등에 간편하게 공유하기 위해** 대시보드 상단의 간편 `[링크 복사]` 유틸리티 버튼을 터치하여 클립보드로 URL을 매끄럽게 가져온다.

### 2.3. 프로필 정보 설정 (인라인 편집 및 토스트 알림)
- 사용자는 **일반 방문자에게 직접 보여질 자신의 실제 이름(`username`)과 자기소개를 새롭게 작성하기 위해** 대시보드 상단 프로필 텍스트 영역 위에 마우스를 올리고 한 번 클릭한다.
- 사용자는 **폼 페이지 전환 등 번거로운 과정 없이 현재 화면에서 내용을 곧바로 변경하기 위해** 활성화된 텍스트 필드(인라인 폼)에 텍스트를 타이핑하여 내용을 수정한다.
- 사용자는 **수정한 프로필 내용을 데이터베이스에 저장하기 위해** 키보드의 `Enter` 키를 누르거나, 입력칸 외부의 빈 영역을 마우스로 클릭(포커스 아웃)한다.
- 사용자는 **정상적으로 서버에 저장이 완료되었음을 직관적으로 인지하기 위해** 화면 모서리에 잠깐 나타났다 부드럽게 사라지는 작은 긍정 피드백 알림창(Toast 메시지: "성공적으로 저장되었습니다" 등)을 눈으로 확인한다.

### 2.4. 새로운 링크 추가 및 최상단 배치
- 사용자는 **새로운 외부 이동 경로 URL을 방문자에게 리스트 형태로 제공하기 위해** 대시보드의 중앙부에 있는 `[+ 링크 추가]` 버튼을 클릭한다.
- 사용자는 **생성될 링크의 기본 데이터를 정의하기 위해** 나타난 폼에 직관적인 링크의 제목(Title)과 실제 주소(URL)를 기입한 뒤에 추가 저장 버튼을 최종 클릭한다.
- 사용자는 **자신이 방금 작성한 항목이 제대로 들어갔는지 빠르게 파악하기 위해** 추가된 링크가 기존 링크 목록의 맨 아래가 아닌 **맨 위(최상단)** 위치에 밀어내기 식으로 즉시 생성되는 모습을 확인한다.
- 사용자는 **별도 이미지 업로드를 들이지 않아도 웹사이트의 대표 이미지가 나타남을 체험하기 위해** 새로 등록된 리스트 항목 옆에 구글 Favicon API를 거쳐 자동으로 가져와진 원래 웹사이트의 파비콘 이미지가 아이콘으로 채워지는 모습을 구경한다.

### 2.5. 기존 링크 수정 (인라인 편집)
- 사용자는 **기존에 등록한 링크의 제목이나 URL에 존재하는 오타를 교정하기 위해** 기존 링크 리스트 항목의 텍스트 부분을 그냥 클릭한다.
- 사용자는 **텍스트 자체가 입력 가능한 인라인 폼으로 변하면 신속하게 내용을 고치기 위해** 타자 입력 혹은 URL 붙여넣기를 진행한다.
- 사용자는 **수정된 링크 내용을 최종 확정하기 위해** 프로필 인라인 수정과 동일하게 `Enter` 키를 누르거나 외부 여백을 클릭하고, 성공을 알리는 토스트 작은 알림창을 통해 정상 반영되었음을 확인한다.

### 2.6. 기존 링크 삭제 (안전장치 경고창)
- 사용자는 **오래되었거나 더 이상 외부에 전시하고 싶지 않은 링크를 제거하기 위해** 리스트 목록에서 해당 링크의 우측 끝에 위치한 붉은색 `[삭제]` 버튼을 터치(또는 클릭)한다.
- 사용자는 **자신의 클릭 조작이 실수가 아님을 서비스 시스템에 답하기 위해** 팝업 형태로 화면 중앙에 표시된 "해당 링크를 정말로 삭제하시겠습니까?" 형태의 방어용 확인창(Confirm Modal 프롬프트)을 마주하게 된다.
- 사용자는 **링크의 영구 삭제를 확정 짓기 위해** 팝업 내 위치한 [확인/삭제] 버튼을 누르며, 그 즉시 링크가 화면 리스트에서 영구적으로 스스륵 사라지고, 안전하게 지워졌다는 토스트 알림 메시지가 뜨는 것을 최종 확인한다.
